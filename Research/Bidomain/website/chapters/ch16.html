Chapter 16

Elliptic Solvers: The Schur Complement

The elliptic equation for $\phi_e$ — the computational bottleneck of the bidomain model, consuming 70–85% of total runtime — produces a symmetric but indefinite saddle-point system. This chapter develops the solver stack: block preconditioners that exploit the 2×2 structure, algebraic multigrid (AMG) for the inner solves, Krylov methods (MINRES, GMRES) for the outer iteration, and null-space handling for the isolated-tissue case.

16.1   The Saddle-Point Structure

The 2×2 block system (15.2) is symmetric but indefinite. This means its eigenvalues include both positive and negative values — a hallmark of saddle-point problems that arise throughout computational physics (Stokes flow, constrained optimization, mixed FEM). The key implications are:

  • Conjugate Gradient (CG) cannot be used on the full system — CG requires positive definiteness.
  • MINRES (Minimum Residual) is the natural Krylov solver for symmetric indefinite systems.
  • GMRES works for any system but requires more memory (stores the full Krylov basis).
  • The condition number of the bidomain system ranges from $10^8$ to $10^{10}$ for realistic meshes — preconditioning is absolutely essential.

16.2   Block Preconditioners

The 2×2 block structure suggests a natural family of preconditioners that exploit the block decomposition rather than treating the system as a monolithic matrix.

Block Diagonal Preconditioner

$$\mathbf{P}_{\text{diag}} = \begin{pmatrix}\mathbf{A}_{11} & \mathbf{0} \\ \mathbf{0} & \mathbf{A}_{22}\end{pmatrix}$$

Application: solve two independent SPD systems ($\mathbf{A}_{11}^{-1}\mathbf{x}_1$ and $\mathbf{A}_{22}^{-1}\mathbf{x}_2$) in each preconditioning step. Simple to implement but ignores the off-diagonal coupling, leading to 50–200 outer iterations.

Block LDU (Triangular) Preconditioner

A more sophisticated choice uses the Schur complement:

$$\mathbf{S} = \mathbf{A}_{22} - \mathbf{A}_{21}\,\mathbf{A}_{11}^{-1}\,\mathbf{A}_{12}$$

The lower-triangular preconditioner is:

$$\mathbf{P}_{\text{LDU}} = \begin{pmatrix}\mathbf{A}_{11} & \mathbf{0} \\ \mathbf{A}_{21} & \mathbf{S}\end{pmatrix}$$

With exact inverses of $\mathbf{A}_{11}$ and $\mathbf{S}$, the preconditioned system converges in at most 2–3 Krylov iterations. In practice, we approximate:

  • $\mathbf{A}_{11}^{-1}$ using one AMG V-cycle for the mass-shifted stiffness matrix.
  • $\mathbf{S}^{-1}$ using one AMG V-cycle for an approximation of the Schur complement (often $\tilde{\mathbf{S}} \approx \mathbf{K}_e$ or $\tilde{\mathbf{S}} \approx \mathbf{K}_i + \mathbf{K}_e$).

With these approximations, convergence typically requires 10–30 outer iterations — a dramatic improvement over unpreconditioned solves.

The Schur complement has a physical interpretation. It represents the "effective" equation for $\phi_e$ after eliminating $V_m$. Computing it exactly is as expensive as solving the original system, but good approximations are cheap: for instance, $\tilde{\mathbf{S}} \approx \mathbf{K}_e$ works well because the mass-matrix contribution in $\mathbf{A}_{11}$ makes $\mathbf{A}_{11}^{-1}\approx (\Delta t/\mathbf{M})$ for small $\Delta t$, and the correction term $\mathbf{K}_i\,(\Delta t/\mathbf{M})\,\mathbf{K}_i$ is small.

16.3   Algebraic Multigrid (AMG)

AMG is the workhorse preconditioner for elliptic and elliptic-like systems in cardiac simulations. It constructs a hierarchy of progressively coarser "meshes" (algebraically, not geometrically) and uses smoothing + coarse-grid correction cycles to damp errors at all spatial frequencies simultaneously.

For the bidomain system, AMG is applied at the sub-block level:

  • $\mathbf{A}_{11}$ block: AMG handles the mass-shifted intracellular stiffness efficiently. Convergence in 1–2 V-cycles.
  • $\mathbf{A}_{22}$ or $\mathbf{S}$ block: AMG handles the combined stiffness (elliptic problem for $\phi_e$). This is the bottleneck — the elliptic solve typically consumes 60–80% of total linear solver time.

16.4   Krylov Methods: MINRES, GMRES, and Friends

The outer Krylov solver iterates on the preconditioned system until the residual falls below a tolerance (typically $10^{-6}$ to $10^{-8}$).

MINRES

The mathematically optimal choice for symmetric indefinite systems. It minimizes the residual norm over the Krylov subspace at each iteration. Requires a symmetric preconditioner (block diagonal qualifies; block LDU does not, unless symmetrized). Memory-efficient: stores only 3 vectors.

GMRES

Works for any system (no symmetry requirement). Allows non-symmetric preconditioners like block LDU. However, memory grows linearly with iteration count (each iteration stores one vector). A restart parameter (typically 30–50) limits memory at the cost of slower convergence.

Flexible GMRES (FGMRES)

A variant that allows the preconditioner to change at each iteration — useful when AMG V-cycles produce slightly different results each time (as happens with some adaptive smoothers). Memory cost is 2× GMRES.

Recommended solver stack for bidomain:
Outer solver: FGMRES(30) or MINRES
Preconditioner: Block LDU with AMG V-cycles for sub-blocks
Expected iterations: 10–30 per time step
Expected speedup over unpreconditioned: 50–100×

16.5   The Null-Space Problem

The elliptic equation for $\phi_e$ with homogeneous Neumann boundary conditions means $\phi_e$ is determined only up to an additive constant — the matrix $\mathbf{A}_{22}$ has a one-dimensional null space spanned by the constant vector $\mathbf{1}$. As discussed in Section 12.6, this is a direct consequence of the isolated-tissue boundary conditions.

Three strategies for handling the null space:

  1. Pin one node: set $(\phi_e)_k = 0$ for some node $k$. Computationally simplest; physically reasonable (choose a "ground" reference). Modify one row/column of $\mathbf{A}_{22}$.
  2. Integral constraint: enforce $\int_\Omega \phi_e\,dx = 0$ via a Lagrange multiplier. Adds one extra equation to the system. More elegant for theoretical analysis.
  3. Deflation: project the constant mode out of the Krylov subspace at each iteration. Algebraically clean; no modification to the matrix. Slightly more complex to implement.
  4. Spectral handling (DCT): in the DCT solver, the $(0,0)$ coefficient corresponds exactly to the constant mode. Setting $\hat{\mathbf{V}}_{0,0} = 0$ after the spectral division is equivalent to pinning the mean to zero — no special matrix modification needed.

Engine V5.4 connection. For monodomain, the null space of $\mathbf{K}$ is removed by the mass matrix in the implicit system (Section 8.7). For the bidomain extension, we recommend pin-one-node as the default strategy, with DCT null-space handling as the automatic approach when using DCTSolver. The DCTSolver._compute_eigenvalues() method already handles the $(0,0)$ singularity by setting the denominator to 1.0 and the solution coefficient to 0.0.

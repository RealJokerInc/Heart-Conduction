Chapter 15

Linear Solvers and Implementation

Every strategy from Chapter 14 ends with one or two linear solves. The parabolic solve is identical to monodomain (Chapter 11). The elliptic solve is new — and it is the computational bottleneck of the entire bidomain simulation. This chapter covers the solvers for both sub-problems, the null-space handling required for isolated-tissue BCs, and the three-tier automatic solver selection that Engine V1 uses to pick the fastest option.

15.1   The Parabolic Sub-Problem: Familiar Territory

The parabolic matrix $\mathbf{A}_{\text{para}} = \frac{1}{\Delta t}\,\mathbf{I} - \theta\,\mathbf{L}_i$ looks exactly like the monodomain implicit system from Chapter 11. It is N×N, symmetric, and positive definite (the $1/\Delta t$ identity dominates). All three solver families from Part II apply directly:

  • PCG (Section 11.3): Preconditioned conjugate gradient. Works on any mesh. With warm-starting (using $V_m^n$ as initial guess), converges in 3–8 iterations.
  • Chebyshev: Fixed iteration count, zero synchronization points. Ideal for GPU. Needs eigenvalue bounds (computed via Gershgorin, cached).
  • Spectral (DCT/DST) (Section 15.3): Direct solve via transform. $O(N \log N)$, exact, no iteration. Requires structured Cartesian grid with uniform BCs per axis.

Because this sub-problem is identical to monodomain, we do not repeat the solver details here. See Chapter 11 for implicit methods, and Appendix C.13 for PCG theory.

15.2   The Elliptic Sub-Problem: The New Challenge

The elliptic matrix $\mathbf{A}_{\text{ellip}} = -(\mathbf{L}_i + \mathbf{L}_e)$ is the combined bulk Laplacian — a pure diffusion operator with no mass/identity term. It has different properties from the parabolic matrix:

PropertyParabolic $\mathbf{A}_{\text{para}}$Elliptic $\mathbf{A}_{\text{ellip}}$
Size$N \times N$$N \times N$
Symmetric?YesYes (with face-based stencil)
Positive definite?Yes (always)Semi-definite with Neumann BCs
Null spaceNoneConstant vector $\mathbf{1}$ (Neumann case)
Condition number$O(1/\Delta t)$$O(N)$ (grid-dependent)

The semi-definiteness means the matrix is singular when all boundaries are Neumann (the isolated-tissue case from Section 12.7). We must handle this null space before solving.

Null-Space Handling

The null space exists because $\phi_e$ is defined only up to an additive constant — adding the same voltage everywhere does not change any gradient or current flow. The engine uses two strategies to remove this ambiguity:

  1. Pin one node: Set $\phi_{e,k} = 0$ for some chosen node $k$. This replaces row $k$ and column $k$ of $\mathbf{A}_{\text{ellip}}$ with the identity (1 on diagonal, 0 elsewhere), and sets $b_k = 0$. The matrix becomes non-singular and SPD. This is the default in Engine V1.
  2. Spectral zero-mode removal: In the DCT solver (Section 15.3), the $(0,0)$ transform coefficient corresponds exactly to the constant mode. Setting $\hat{u}_{0,0} = 0$ after spectral division pins the spatial mean to zero — no matrix modification needed.

Other approaches exist in the literature — Lagrange multipliers (enforce $\int \phi_e\,dx = 0$) and Krylov deflation (project the constant mode out at each iteration) — but they add complexity without practical benefit for the cardiac setting. Engine V1 uses pinning for iterative solvers and zero-mode removal for spectral solvers.

Pinning = choosing sea level. Just as altitude is meaningless without a reference point, $\phi_e$ is meaningless without a reference voltage. Pinning one node to zero is like declaring "this is sea level." All other $\phi_e$ values are then measured relative to this reference. The choice of which node to pin does not affect any physical quantity — only $\phi_e$ differences matter, not absolute values.

15.3   Three-Tier Automatic Solver Selection

Not all bidomain problems are equally hard. A simple isotropic simulation on a Cartesian grid can be solved exactly with a spectral method in $O(N \log N)$. An anisotropic simulation with mixed boundary conditions requires iterative solvers. Engine V1 automatically selects the fastest solver based on the problem's properties:

TierSolverWhen usedCost
Tier 1 Direct spectral (DCT/DST/FFT) Isotropic + structured Cartesian + uniform BCs per axis $O(N \log N)$, exact
Tier 2 PCG with spectral preconditioner Anisotropic + structured Cartesian + uniform BCs per axis $O(k \cdot N \log N)$, $k \approx 5\text{–}15$
Tier 3 PCG (Jacobi preconditioner) Unstructured mesh, mixed BCs, or general geometry $O(k \cdot N)$, $k \approx 20\text{–}50$

The selection logic inspects two properties of the problem:

  1. Is the grid Cartesian with uniform BCs per axis? If yes, spectral transforms (DCT for Neumann, DST for Dirichlet, FFT for periodic) can diagonalize the Laplacian. This enables Tier 1 or Tier 2.
  2. Is the conductivity isotropic? If yes, a single scalar $D = D_i + D_e$ characterizes the elliptic operator, and the spectral solver gives an exact answer (Tier 1). If the conductivity is anisotropic (different in $x$ and $y$), the spectral solver can still serve as a preconditioner for PCG (Tier 2) — it captures the dominant structure, and PCG handles the anisotropic correction in a few iterations.

Think of it like choosing a tool for a job. If the problem is perfectly regular (isotropic, Cartesian), use a precision instrument (spectral — exact, fast). If it is slightly irregular (anisotropic but still Cartesian), use the precision instrument as a starting point and refine with a general-purpose tool (spectral-preconditioned PCG). If it is fully irregular (unstructured mesh, complex geometry), fall back to the general-purpose tool alone (PCG).

Engine V1 connection. The auto-selection logic lives in BidomainSimulation._auto_select_elliptic_solver() in bidomain.py. It inspects the BoundarySpec protocol to determine if BCs are spectral-eligible, and the conductivity to determine if the problem is isotropic. The spectral solver (SpectralSolver in linear_solver/spectral.py) supports per-axis BC selection: bc_x='neumann', bc_y='dirichlet', etc.

Linear solver selection uses a separate factory: 'pcg', 'chebyshev', 'spectral' (Tier 1), 'pcg_spectral' (Tier 2), or the automatic three-tier selection described above.

Key design principle: zero allocation per step. All diffusion solvers pre-allocate workspace tensors ($\mathbf{A}_{\text{para}}$, $\mathbf{B}_{\text{para}}$, temporary vectors) at construction time. The step() method performs only matrix-vector products and linear solves — no memory allocation, no Python object creation. This is critical for GPU performance.

Chapter 15

Parabolic Solvers: The $A_{11}$ Block

After operator splitting separates the parabolic and elliptic halves of the bidomain system, the $V_m$ update requires solving a linear system with the $A_{11}$ block — a symmetric positive definite (SPD) matrix that looks exactly like the monodomain stiffness-plus-mass matrix. This chapter covers the three main solver families for this sub-problem: preconditioned conjugate gradient (PCG), Chebyshev iteration, and spectral (transform-based) methods.

15.1   Preconditioned Conjugate Gradient (PCG)

Before tackling the full bidomain block system, let us master the solver for a single SPD sub-block. The Preconditioned Conjugate Gradient method is the most widely used iterative solver for symmetric positive-definite systems — and in bidomain solvers, it is typically applied to the Schur complement block $\mathbf{S}$ (or to $A_{11}$ during the parabolic substep).

Appendix A.10 developed the theory from first principles: minimizing the energy functional, the conjugate direction construction, and preconditioning. Here we focus on the practical details that make PCG effective for cardiac simulation.

The Algorithm

The full PCG algorithm with Jacobi preconditioning is given in Appendix A.10 (the boxed algorithm). Each iteration requires:

  • One sparse matrix-vector product ($A\mathbf{p}$) — the dominant cost
  • Two inner products ($\mathbf{r}^T\mathbf{z}$ and $\mathbf{p}^T A\mathbf{p}$) — the synchronization bottleneck on GPUs
  • Three vector updates (AXPY operations) — negligible cost

For a 2D cardiac grid with $N$ nodes, the sparse matrix has $O(N)$ nonzero entries (5 per row for FDM, slightly more for FEM), so the matrix-vector product costs $O(N)$. Total cost per PCG solve: $O(k \cdot N)$ where $k$ is the iteration count.

Warm-Starting: The Key to Cardiac Performance

In cardiac simulation, we solve a linear system at every time step. The transmembrane voltage field $V_m$ changes only slightly between consecutive steps (the wavefront advances by roughly $\Delta t \times \text{conduction velocity} \approx 0.02\,\text{ms} \times 60\,\text{cm/s} = 0.0012\,\text{cm}$, which is a fraction of a grid spacing). This means the solution at time $t^{n+1}$ is very close to the solution at time $t^n$.

Warm-starting exploits this by using the previous time step's solution as the initial guess for the current step. Instead of starting from $\mathbf{x}_0 = \mathbf{0}$ (cold start), we set $\mathbf{x}_0 = \mathbf{x}^{n-1}$. The initial residual is already small, and PCG converges in just a few iterations.

Starting strategyTypical iterations (monodomain, CN)Typical iterations (bidomain, Schur)
Cold start ($\mathbf{x}_0 = \mathbf{0}$)20–5040–100
Warm start ($\mathbf{x}_0 = \mathbf{x}^{n-1}$)3–88–20

Warm-starting is the single biggest performance optimization for cardiac PCG. It is free (just keep the previous solution in memory), requires no mathematical sophistication, and typically reduces the iteration count by a factor of 5–10. Always use it.

Convergence Criterion

The standard stopping criterion is a relative residual test: stop when $\|\mathbf{r}_k\|/\|\mathbf{b}\| < \varepsilon$, where $\varepsilon$ is a tolerance (typically $10^{-8}$ for cardiac simulations). This ensures the error is small relative to the forcing. If $\|\mathbf{b}\|$ is near zero (e.g., during a quiescent phase when no wavefront is active), an absolute tolerance fallback prevents unnecessary iterations.

15.2   Chebyshev Polynomial Solver

PCG is optimal in the mathematical sense (it minimizes the error in the $A$-norm over the Krylov subspace), but it has a practical weakness on GPUs: each iteration requires two inner products (global reductions), which force all GPU threads to synchronize. On modern hardware with thousands of cores, this synchronization is a significant bottleneck — the GPU must wait for every thread to finish before the reduction can complete.

The Chebyshev polynomial solver eliminates this bottleneck entirely. Instead of adaptively choosing step sizes (which require inner products), it uses a predetermined sequence of steps based on Chebyshev polynomials. The iteration count is fixed in advance, and no convergence checks occur during the solve. This makes it ideal for GPU execution: every iteration is a pure matrix-vector product plus vector updates — no synchronization.

The Key Idea

A Chebyshev polynomial $T_d(x)$ has the remarkable property that it grows as slowly as possible outside the interval $[-1, 1]$ while satisfying $|T_d(x)| \leq 1$ for $x \in [-1, 1]$. If we map the eigenvalue interval $[\lambda_{\min}, \lambda_{\max}]$ of $A$ onto $[-1, 1]$ and construct a polynomial $p_d(A)$ that approximates $A^{-1}$ (in the sense that $p_d(\lambda) \approx 1/\lambda$ for $\lambda \in [\lambda_{\min}, \lambda_{\max}]$), then $\mathbf{x} \approx p_d(A)\,\mathbf{b}$ is an approximate solution.

The Chebyshev iteration evaluates this polynomial via a 3-term recurrence, which requires only the previous two iterates — no storage of the full iteration history (unlike GMRES).

Eigenvalue Estimation via Gershgorin

The Chebyshev solver needs bounds $[\lambda_{\min}, \lambda_{\max}]$ on the eigenvalues of $A$. Computing exact eigenvalues would be as expensive as solving the system, so we use the Gershgorin circle theorem: every eigenvalue of $A$ lies in at least one disc centered at a diagonal entry $a_{ii}$ with radius equal to the sum of absolute off-diagonal entries in that row. The bounds are:

$$\lambda_{\min} \geq \min_i\left(a_{ii} - \sum_{j \neq i} |a_{ij}|\right), \qquad \lambda_{\max} \leq \max_i\left(a_{ii} + \sum_{j \neq i} |a_{ij}|\right)$$

This requires a single pass over the sparse matrix — $O(N)$ work — and produces conservative (over-estimated) bounds. A safety margin of 10% is typically added to guard against underestimation.

15.3   Spectral Solvers: DCT, DST, and FFT

For structured Cartesian grids (the default in FDM and FVM), there is an entirely different approach to solving the linear system — one that avoids iteration altogether. Instead of gradually improving an approximate solution, a spectral solver transforms the problem into a space where the solution can be written down in closed form, then transforms back. The total cost is just two FFT-like transforms: $O(N \log N)$.

The mathematical foundation is in Appendix A.8–A.9. Here we focus on the three spectral solvers relevant to cardiac simulation and when to use each.

15.3.1   DCT for Neumann Boundary Conditions

The most important case for cardiac EP. Neumann (zero-flux) BCs are the physically correct default for isolated tissue (Section 8.7, Section 12.6). As shown in Appendix A.9, the cosine basis automatically satisfies $\partial u/\partial n = 0$ at the boundaries, so the Discrete Cosine Transform diagonalizes the discrete Laplacian under Neumann BCs.

The algorithm for solving $A_{\text{lhs}}\,\mathbf{V}^{n+1} = \mathbf{b}_{\text{rhs}}$ (from the implicit diffusion step) is:

  1. Reshape: Convert the $N$-vector $\mathbf{b}$ into an $N_x \times N_y$ grid.
  2. Forward 2D DCT: $\hat{\mathbf{b}} = \text{DCT}_{2D}(\mathbf{b}_{\text{grid}})$. Cost: $O(N \log N)$.
  3. Spectral division: $\hat{\mathbf{V}}_{i,j} = \hat{\mathbf{b}}_{i,j}\,/\,d_{i,j}$ where $d_{i,j}$ is the $(i,j)$-th entry of the spectral denominator (see below).
  4. Null-space handling: Set $\hat{\mathbf{V}}_{0,0} = 0$ (the mean-zero convention for Neumann). This is equivalent to pinning in physical space (Section 12.6).
  5. Inverse 2D DCT: $\mathbf{V}_{\text{grid}} = \text{IDCT}_{2D}(\hat{\mathbf{V}})$. Cost: $O(N \log N)$.
  6. Flatten: Convert $N_x \times N_y$ grid back to $N$-vector.

The spectral denominator $d_{i,j}$ depends on the time-stepping scheme (from Chapters 9–10):

(15.1) $$d_{i,j} = \begin{cases} \chi C_m - \tfrac{1}{2}\Delta t\,D\,\lambda_{i,j} & \text{Crank-Nicolson (CN)} \\ \chi C_m - \Delta t\,D\,\lambda_{i,j} & \text{BDF1 (Backward Euler)} \\ 3\chi C_m - 2\Delta t\,D\,\lambda_{i,j} & \text{BDF2} \end{cases}$$

where $\lambda_{i,j}$ are the 2D DCT eigenvalues from equation (A.10):

$$\lambda_{i,j} = \frac{2}{\Delta x^2}\left(\cos\!\left(\frac{\pi i}{N_x}\right) - 1\right) + \frac{2}{\Delta y^2}\left(\cos\!\left(\frac{\pi j}{N_y}\right) - 1\right)$$

Since all $\lambda_{i,j} \leq 0$ and $D > 0$, the denominator $d_{i,j}$ is always positive (except at $(i,j) = (0,0)$, which is handled by the null-space step). The division is safe and numerically stable.

15.3.2   DST for Dirichlet Boundary Conditions

When the voltage is clamped at the boundary (Dirichlet BCs), the sine basis is appropriate. As shown in Appendix A.9, $\sin(k\pi x/L)$ vanishes at both boundaries, so the Discrete Sine Transform diagonalizes the Laplacian under Dirichlet conditions.

The algorithm is identical to the DCT case, with two changes: (1) replace DCT with DST (and IDCT with IDST), and (2) use the DST eigenvalues from equation (A.9). There is no null-space issue — the DST eigenvalues are all strictly negative, so the denominator is always positive.

In cardiac simulation, pure Dirichlet BCs on the entire boundary are uncommon (they would mean voltage-clamping the entire tissue surface). The DST solver is more relevant for sub-problems where Dirichlet conditions arise — for instance, when solving the Schur complement with a grounded-bath setup (Section 12.6).

15.3.3   Combining DCT and DST for Mixed BCs

When different axes have different boundary conditions — for example, Neumann on the $x$-boundaries and Dirichlet on the $y$-boundaries — we apply a cosine transform along $x$ and a sine transform along $y$. The 2D transform is:

$$\hat{\mathbf{b}}_{i,j} = \text{DCT}_x\bigl[\text{DST}_y\bigl[\mathbf{b}_{\text{grid}}\bigr]\bigr]$$

and the eigenvalues combine the DCT formula for $x$ with the DST formula for $y$:

$$\lambda_{i,j} = \underbrace{\frac{2}{\Delta x^2}\left(\cos\!\left(\frac{\pi i}{N_x}\right) - 1\right)}_{\text{Neumann in }x} + \underbrace{\frac{2}{\Delta y^2}\left(\cos\!\left(\frac{\pi j}{N_y+1}\right) - 1\right)}_{\text{Dirichlet in }y}$$

Recall the key limitation from Appendix A.9: this works only when each axis has a single, uniform BC type. You cannot use spectral methods when one end of the $x$-axis has Neumann and the other has Dirichlet — in that case, fall back to PCG (Section 15.1) or Chebyshev (Section 15.2).

15.3.4   FFT for Periodic Boundary Conditions

For completeness, the standard FFT solver handles periodic BCs. This is rarely used in cardiac simulation (tissue is not periodic), but it appears in benchmark problems and in simplified models of re-entrant spiral waves on a periodic domain. The eigenvalues are:

$$\lambda_{i,j} = -\left(\frac{2\pi\,\text{fftfreq}(i, N_x)}{\Delta x}\right)^2 - \left(\frac{2\pi\,\text{fftfreq}(j, N_y)}{\Delta y}\right)^2$$

where $\text{fftfreq}(k, N)$ returns the $k$-th frequency bin. The algorithm uses torch.fft.fft2 / ifft2.

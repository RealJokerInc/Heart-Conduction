DECISION 1: GPU or CPU Target?
═══════════════════════════════

GPU (recommended for cardiac):
  Library: NVIDIA AMGX
  Performance: 2-5× faster than CPU
  Integration: C API, requires CUDA
  Cardiac validation: Excellent (proven on bidomain)
  Go to: README.md → Available Libraries → For Production Use

CPU or Flexible:
  Library: Hypre BoomerAMG via PETSc
  Performance: Baseline reference (faster on recent systems)
  Integration: Mature, open-source, standard
  Cardiac validation: Excellent (openCARP standard)
  Go to: README.md → Available Libraries → For Production Use

DECISION 2: Which Preconditioner?
══════════════════════════════════

Simple (quick prototype):
  Preconditioner: Block Diagonal
  Performance: 30-50 iterations
  Dev time: Fastest
  Go to: BIDOMAIN_LINEAR_SOLVERS.md → Section 2.1

Recommended (production):
  Preconditioner: Block LDU
  Performance: 15-30 iterations
  Dev time: Moderate
  Go to: BIDOMAIN_LINEAR_SOLVERS.md → Section 2.2-2.3

Advanced (optimized):
  Preconditioner: FGMRES + SIMPLE
  Performance: 10-20 iterations
  Dev time: Longest
  Go to: BIDOMAIN_LINEAR_SOLVERS.md → Section 4.4, 6.4

DECISION 3: Which Krylov Method?
════════════════════════════════

GMRES:
  Pros: Standard, well-known
  Cons: Memory-intensive (m vectors)
  Recommended when: Small problems only
  Go to: BIDOMAIN_LINEAR_SOLVERS.md → Section 4.1

MINRES (STRONGLY RECOMMENDED):
  Pros: Memory-efficient, exploits symmetry
  Cons: Slightly less standard
  Recommended when: Large problems, GPU
  Go to: BIDOMAIN_LINEAR_SOLVERS.md → Section 4.2

COMMON QUESTIONS & ANSWERS
================================================================================

Q: Why is bidomain so much harder than monodomain?
A: See BIDOMAIN_LINEAR_SOLVERS.md Section 1.3
   Quick: κ grows as O(h⁻⁴) instead of O(h⁻²)

Q: Can I just use my existing CG solver?
A: NO! CG will diverge on indefinite systems.
   See BIDOMAIN_LINEAR_SOLVERS.md Section 4 for why.
   Use MINRES instead (similar convergence, works for indefinite).

Q: Which library should I use?
A: Use decision tree in RESEARCH_SUMMARY.txt
   → GPU? Use AMGX
   → CPU production? Use Hypre BoomerAMG
   → Need to modify internals? Use PyAMG (small problems only)

Q: How many iterations should I expect?
A: 15-30 iterations with Block LDU + AMG
   If >50 iterations, something is wrong.
   See README.md performance table.

Q: Can I use my V5.4 Chebyshev solver?
A: Yes, but only for the φe solve (elliptic part).
   See BIDOMAIN_LINEAR_SOLVERS.md Section 5.5

Q: Should I use one or multiple Gauss-Seidel iterations?
A: Usually one is sufficient! See Section 6.2
   Research shows ρ(G) ~ 0.3-0.5, so one iteration fixes 50% error.

Q: What's the memory overhead compared to monodomain?
A: 2× larger system (2N vs. N unknowns)
   Plus AMG hierarchy (typically 20-30% overhead)
   Total: ~2.5× memory vs. monodomain

Q: Can I run on a single GPU?
A: Yes, up to ~50-100 million unknowns.
   For larger: multi-GPU (AMGX supports distributed MPI)

================================================================================
TROUBLESHOOTING CHECKLIST
================================================================================

If solver diverges:
  ☐ Is matrix symmetric? Check A12 = A21ᵀ
  ☐ Are you using CG? (Switch to MINRES)
  ☐ Is preconditioner applied correctly?
  ☐ Check matrix assembly: A11, A22 should be SPD

If solver converges slowly (>100 iterations):
  ☐ Is AMG preconditioner active?
  ☐ Check AMG setup time (should be 1-2 seconds)
  ☐ Verify Chebyshev smoother configured (not ILU)
  ☐ Try tighter tolerance (rtol=1e-4) first, then tighten

If out of memory:
  ☐ Using GMRES with large restart? Switch to MINRES
  ☐ Is AMG preconditioner memory reasonable? (~2× matrix size)
  ☐ Are you storing full matrix or doing matrix-free?

If slow per-iteration:
  ☐ Is SpMV optimized? Check GPU utilization (70%+)
  ☐ Are you fusing kernels (SpMV + precond)?
  ☐ Check data types: FP32 vs FP64

  ---
  OpenCarp / Monodomain Solver Implementation - Research Findings

  1. The Monodomain Equation

  The governing PDE is:

  χ(Cm ∂V/∂t + Iion(u,V)) = ∇·(σ∇V) + Istim
  ∂u/∂t = f(u,V)

  Where:
  - V = transmembrane voltage (mV)
  - u = vector of ionic state variables (gating, concentrations)
  - σ = conductivity tensor (mS/cm)
  - χ = surface-to-volume ratio (~1400 cm⁻¹)
  - Cm = membrane capacitance (1 µF/cm²)
  - Iion = ionic current from cell model (µA/µF)

  2. Operator Splitting Approach

  The standard approach separates reaction (ionic ODEs) from diffusion (PDE):

  Godunov (1st order) splitting:
  For each time step Δt:
    1. Solve ionic step: Cm·dV/dt = -Iion(V,u), du/dt = f(V,u)  for Δt
    2. Solve diffusion: Cm·dV/dt = (1/χ)∇·(σ∇V)  for Δt

  Strang (2nd order) splitting:
  For each time step Δt:
    1. Diffusion half-step: Δt/2
    2. Ionic full-step: Δt
    3. Diffusion half-step: Δt/2

  Key insight from https://pmc.ncbi.nlm.nih.gov/articles/PMC4519349/: "the ionic part of the split is represented by equations that are spatially decoupled, local, ordinary differential equations" - enabling node-by-node integration without global communication.

  3. Spatial Discretization Options

  Finite Difference (simpler, what we should use):

  5-point stencil for 2D Laplacian:
  ∇²V ≈ (V[i+1,j] + V[i-1,j] + V[i,j+1] + V[i,j-1] - 4·V[i,j]) / dx²

  9-point stencil (better isotropy):
  ∇²V ≈ (4·(V[i±1,j] + V[i,j±1]) + V[i±1,j±1] - 20·V[i,j]) / (6·dx²)

  Finite Element (openCARP approach):
  - More complex but handles irregular geometries
  - Uses mass matrix lumping for efficiency
  - Galerkin discretization with various element types

  4. Time Stepping

  For diffusion (semi-implicit preferred for stability):
  - Crank-Nicolson: unconditionally stable, 2nd order accurate
  - Backward Euler: unconditionally stable, 1st order

  For ionic ODEs (at each node):
  - Rush-Larsen for gating variables (already implemented!)
  - Forward Euler or adaptive sub-stepping for concentrations
  - Sub-stepping with dt_ionic = 0.001-0.01 ms common

  Stability condition (explicit diffusion):
  dt ≤ dx² · Cm · χ / (2·D·σ)    # CFL-like condition

  For implicit: no stability restriction, but accuracy requires small dt.

  5. Boundary Conditions

  No-flux (Neumann) at tissue boundaries:
  ∂V/∂n = 0   (normal derivative = 0)

  Implementation via ghost cells:
  # Copy interior values to ghost cells
  V[0,:] = V[1,:]      # left boundary
  V[-1,:] = V[-2,:]    # right boundary  
  V[:,0] = V[:,1]      # bottom
  V[:,-1] = V[:,-2]    # top

  6. Key Implementation Considerations

  From https://pmc.ncbi.nlm.nih.gov/articles/PMC4525671/:
  - Element sizes >200 µm produce questionable conduction velocities
  - Typical spatial resolution: 100-200 µm for cardiac tissue
  - The cell Thiele modulus ϕc = h·√(k/D) should be ≲ 1.0 for numerical stability

  From https://cac.cornell.edu/myers/teaching/ComputationalMethods/ComputerExercises/CardiacDynamics/CardiacDynamics.html:
  - Use array slicing for efficient Laplacian computation (vectorized, no loops)
  - Ghost cell approach for boundary conditions

  7. Recommended Implementation Strategy for Phase 2

  Based on these findings, I recommend:

  # Pseudocode for monodomain solver

  def step_monodomain(V, states, dt, dx, D, model):
      """One time step of monodomain solver using operator splitting."""

      # 1. Apply boundary conditions (no-flux)
      apply_ghost_cells(V)

      # 2. Ionic step at each node (can be parallelized)
      for each node (i,j):
          Istim = get_stimulus(i, j, t)
          Iion, states[i,j] = model.ionic_step(V[i,j], states[i,j], dt, Istim)
          V[i,j] -= dt * Iion / Cm  # Voltage update from ionic currents

      # 3. Diffusion step (can use explicit or implicit)
      # Explicit (simple but requires small dt):
      laplacian_V = compute_laplacian(V, dx)  # 5-point stencil
      V += dt * D / (Cm * chi) * laplacian_V

      return V, states
